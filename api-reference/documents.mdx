---
title: 'Get Document'
description: 'Retrieve a fresh signed URL for an existing PDF'
api: 'GET /v1/documents/{id}'
---

## Request

<ParamField header="x-api-key" type="string" required>
  Your API key. Starts with `pk_live_`.
</ParamField>

<ParamField path="id" type="string" required>
  The document ID returned from the render endpoint.
</ParamField>

## Response

<ResponseField name="id" type="string">
  The document ID.
</ResponseField>

<ResponseField name="url" type="string">
  Fresh signed download URL. Expires after 1 hour.
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 timestamp when the signed URL expires.
</ResponseField>

<RequestExample>
```bash
curl -s https://api.renderbase.app/v1/documents/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "x-api-key: pk_live_YOUR_KEY"
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "url": "https://r2.example.com/signed?token=xyz",
  "expires_at": "2026-03-01T16:00:00.000Z"
}
```
</ResponseExample>

## Errors

| Status | Code | Description |
|--------|------|-------------|
| 404 | `NOT_FOUND` | Document does not exist or belongs to a different API key |

## Notes

- Signed URLs expire after **1 hour**. Call this endpoint again to get a fresh URL.
- Documents are scoped to your API key. You cannot access another user's documents.
- Documents are stored indefinitely. Only the signed URL expires, not the document itself.
